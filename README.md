> Коротко: Консольный прототип ролевой игры в стиле Dungeons & Dragons, реализованный на принципах объектно‑ориентированного программирования (OOP) на Python.

---

## Что это

Этот репозиторий содержит исходники прототипа DnD‑подобной игры. Проект сделан в учебных/экспериментальных целях — чтобы отработать архитектуру на классах, механику сражений, взаимодействие с инвентарём и простую игровую логику.

---

## Основные возможности (текущее состояние)

* Боевая система (пошаговая).
* Получение лута с побеждённых монстров (looting).
* Покупка/продажа предметов (простая экономика магазина).
* Костёр/отдых для полного восстановления HP.
* Модульная структура: классы персонажей, игровые функции и ассеты разделены по папкам.

---

## Быстрый старт

1. Убедитесь, что у вас установлен Python 3.8+ (рекомендуется 3.10+).
2. Склонируйте репозиторий:

bash
git clone https://github.com/Ilay4133/Dnd-like_game_on_OOP_Python.git
cd Dnd-like_game_on_OOP_Python

3. Если появится файл requirements.txt, установите зависимости:

bash
pip install -r requirements.txt

4. Запустите игру:

bash
python Main.py

> Примечание: пока проект — консольный прототип. Если в коде будут внешние зависимости (например, pygame), они будут указаны в requirements.txt.

## Структура проекта

Dnd-like_game_on_OOP_Python/
├─ .idea/                # настройки IDE (необязательно)
├─ Classes/              # классы персонажей, NPC, предметов
├─ Game_functions/       # логика боя, торговли, сохранений и пр.
├─ Assets.py             # описания предметов/ресурсов (списки, словари)
├─ Main.py               # точка входа — запускает игровой цикл
├─ README.md             # этот файл

---

## Как читать код

* Папка Classes содержит определения сущностей: игроки, враги, предметы. Именно здесь инкапсулируется состояние и поведение (HP, атаки, баффы и т.д.).
* В Game_functions вынесены отдельные процессы: пошаговая логика боя, магазин, система лута, процедуры отдыха/восстановления.
* Assets.py — простая база данных предметов и стартовых значений, используемых при создании сущностей.
* Main.py собирает всё вместе и запускает пользовательский цикл (создание персонажа, встреча с врагами, интерфейс выбора действий).
